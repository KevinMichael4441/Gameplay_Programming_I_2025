# Gameplay Programming I 2025 Project by Kevin Michael

This game is made entirely in C. The NPC and the player are driven by a Finite State Machine (FSM). Input is handled for keyboard as well as controllers with the help of the 'command pattern' design. 

For the player, input is read and appropriate command is set. Based on active command, the mediator sends out events for the FSM to handle states. The process is simlar to the NPC with the exception of commands being set algorithmically rather than via input.
